# UI Project Template

这个模板是按照你当前 `ui` 子页面里“一个完整项目”的展示逻辑拆出来的。

## 适用场景

- 你想新增一个 UI 项目
- 这个项目包含 1 个项目主介绍 + 多个往下延展的内容模块
- 希望以后只替换文字和图片，不再从头搭结构

## 推荐使用方式

1. 复制整个 `ui-project-template` 文件夹
2. 重命名为新的英文目录名
   - 例如：`ui-project-02`
   - 例如：`process-system`
3. 打开新的 `index.html`
4. 按顺序替换：
   - 项目标题
   - 副标题
   - 左侧蓝色说明卡
   - 右侧主视觉区
   - 下面每个内容模块的标题、文案、图片
5. 把你的图片放进对应目录，或者直接引用现有图片
6. 在 GitHub Desktop 提交并推送

## 后续如何继续加内容

如果这个项目需要更多展示内容，直接在页面里继续复制这一整段：

`<section class="section"> ... </section>`

然后改成新的：
- 标签
- 模块标题
- 说明文字
- 图片

## 命名建议

- 用英文小写目录名
- 不要空格
- 不要中文路径

## 图片建议

- 大图尽量压缩后再放进去
- 手机端体验更好
- 单张图尽量控制在 300KB 到 600KB 左右
